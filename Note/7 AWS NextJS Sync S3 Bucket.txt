Production setup for **Next.js static assets offloading to Amazon S3** during deployment.

You want:

* Build Next.js app
* Upload generated `.js`, `.css`, chunks, images from `.next/static`
* Store in **s3://s3-nextjs-bucket**
* Serve assets from S3 / CloudFront

For the S3 bucket "s3-nextjs-bucket", web url to access it where region "ap-south-1"
	https://s3-nextjs-bucket.s3.ap-south-1.amazonaws.com/

--delete is an AWS CLI sync option used with:

aws s3 sync source destination --delete

It means:
	Delete files in the destination (S3 bucket) that no longer exist in the source folder.

✅ Example
	Suppose your local build folder:

	.next/static/
 	├── main.js
	├── style.css

	And S3 bucket currently has:

	_next/static/
 	├── main.js
 	├── style.css
 	├── old.js

Run:
	aws s3 sync .next/static s3://s3-nextjs-bucket/_next/static --delete
	Result in S3:
	_next/static/
 	├── main.js
 	├── style.css

	✅ old.js gets removed.
